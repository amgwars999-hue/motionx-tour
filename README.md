## 🌐 صفر تا صد: راه‌اندازی روی دامنه‌ی واقعی

### روش ۱: ساده‌ترین راه — GitHub Pages (رایگان، ۵ دقیقه)

```bash
# ۱. یه ریپو روی github.com بساز به نام motionx-tour
# ۲. توی ترمینال:
cd motionx-tour
git init
git add .
git commit -m "Motion X virtual tour"
git branch -M main
git remote add origin https://github.com/USERNAME/motionx-tour.git
git push -u origin main

# ۳. توی GitHub: Settings > Pages > Source: main branch > Save
# سایتت میاد بالا روی:
# https://USERNAME.github.io/motionx-tour/
```

### روش ۲: Vercel (حرفه‌ای، رایگان، با دامنه‌ی اختصاصی)

1. برو به [vercel.com](https://vercel.com) و با گیت‌هاب login کن
2. کلیک "Add New Project" → ریپوی motionx-tour رو انتخاب کن
3. کلیک Deploy. تموم!
4. توی Settings → Domains، دامنه‌ی خودت (مثل `motionx.ir`) رو اضافه کن

### روش ۳: هاست ایرانی (لیارا / آروان / فرابرد)

برای کاربرای ایرانی سریع‌تره. فایل‌ها رو با FTP یا پنل آپلود کن. این پروژه **استاتیک خالصه** و نیاز به Node.js یا سرور نداره.

```
upload: index.html + public/ (تمام فایل‌ها)
```

## 🏗️ مرحله بعدی: ارتقا به نسخه‌ی فوتورئالیستی

این نسخه‌ای که الان داری یه **MVP عالی** برای راه‌اندازی سریعه. برای ارتقا به سطح بعد:

### مرحله A: تور 360 از فضای واقعی Motion X

تصاویر و ویدیوهایی که داری رو می‌تونیم اضافه کنیم:

1. **عکس‌های 360 پانورامی** اگه داری → روی هر زون به‌جای billboard به یه عکس کروی تبدیل می‌شه
2. **ویدیوهای معرفی بازی** → روی صفحه‌های مجازی داخل هر زون پلی می‌شن (با `VideoTexture` در Three.js)

تصاویرت رو بذار توی پوشه‌ی `public/images/` و آدرس‌شون رو بهم بده تا توی کد جایگزین کنم.

### مرحله B: اسکن فضای واقعی (فوتورئالیستی)

برای این که کاربر حس کنه واقعاً تو Motion X راه می‌ره:

#### گزینه‌ی ارزان و سریع: Polycam یا Luma AI
- اپ موبایل **Polycam** رو نصب کن
- با iPhone (با LiDAR) یا گوشی اندروید دور فضا فیلم بگیر
- خروجی **Gaussian Splat** (`.splat` یا `.ply`) بگیر
- توی Three.js با کتابخونه‌ی [`@mkkellogg/gaussian-splats-3d`](https://github.com/mkkellogg/GaussianSplats3D) لود کن

#### گزینه‌ی حرفه‌ای: Matterport
- دستگاه **Matterport Pro3** اجاره کن (~۲ تا ۵ میلیون تومن یک‌روزه)
- خروجی رو روی Matterport Cloud هاست کن
- لینکش رو embed کن، یا با Matterport SDK سفارشی‌سازی کن

### مرحله C: تجربه‌های پیشرفته

- 🎙️ **راهنمای صوتی AI** با ElevenLabs (کاربر سوال بپرسه، صدا جواب بده)
- 👥 **Multiplayer** با PartyKit (چند کاربر همزمان توی فضا)
- 🥽 **WebXR / VR** (کاربر با Meta Quest وارد بشه)
- 📱 **اپ موبایل** با Capacitor از همین کد
- 🎯 **سیستم رزرو آنلاین** متصل به CRM
- 📊 **Analytics** که ببینی کاربرا کجاها بیشتر می‌رن

## 🎵 درباره‌ی صدای 8D

این پروژه از **Web Audio API** + **HRTF (Head-Related Transfer Function)** استفاده می‌کنه که:
- موزیک رو دور سر کاربر می‌چرخونه (همون افکت معروف 8D یوتیوب)
- صدای هر زون از موقعیت واقعی‌اش پخش می‌شه
- وقتی کاربر سرشو می‌چرخونه، صدا هم می‌چرخه (تعاملی، نه pre-rendered)

**نکته:** برای فایل MP3 موزیک واقعی، می‌تونی توی پوشه‌ی `public/audio/` فایل بذاری و توی `initAudio()` به جای oscillator‌ها از این استفاده کنی:

```js
const audio = new Audio('public/audio/your-track.mp3');
audio.loop = true; audio.crossOrigin = 'anonymous';
const source = audioCtx.createMediaElementSource(audio);
source.connect(filter); // به جای oscillator
audio.play();
```

### راه‌اندازی Local Server (اگه دابل‌کلیک کار نکرد):

```bash
# با Python:
cd motionx-tour
python3 -m http.server 8000
# باز کن: http://localhost:8000

# یا با Node:
npx serve .
```
